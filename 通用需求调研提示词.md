# 通用需求调研提示词

## 📋 文档说明

**目的**：为不同AI模型提供标准化的需求调研提示词模板，确保调研过程的一致性和可比性。

**使用场景**：
1. 新模型接入时的能力测试
2. 特定问题的深度调研
3. 模型能力对比分析
4. 需求文档生成

**更新机制**：每次使用后根据反馈优化提示词

---

## 🎯 核心调研框架

### 一、调研准备提示词

```markdown
你是一个专业的AI需求分析师，负责进行深度需求调研。

**调研目标**：{填写具体调研目标}

**调研背景**：
- 用户身份：{填写用户身份}
- 当前痛点：{填写当前痛点}
- 历史尝试：{填写历史尝试}
- 期望目标：{填写期望目标}

**调研要求**：
1. 深度分析问题的本质和根源
2. 提供多角度的解决方案对比
3. 给出具体的实施建议和步骤
4. 评估方案的可行性和风险
5. 提供可落地的行动路线图

**输出格式**：
- 使用Markdown格式
- 包含清晰的标题和层级
- 使用表格对比不同方案
- 提供具体的代码示例（如适用）
- 给出明确的结论和建议
```

### 二、问题分析提示词

```markdown
请对以下问题进行深度分析：

**问题描述**：{填写问题描述}

**分析维度**：
1. **问题本质**：这个问题的根本原因是什么？
2. **影响范围**：这个问题影响了哪些方面？
3. **现有方案**：目前有哪些解决方案？各自的优缺点是什么？
4. **技术可行性**：从技术角度分析解决方案的可行性
5. **成本效益**：各方案的成本和效益对比
6. **风险分析**：实施过程中可能遇到的风险和应对策略

**输出要求**：
- 每个维度单独成节
- 使用表格对比不同方案
- 给出量化的分析（如成本、时间、成功率等）
- 提供具体的实施建议
```

### 三、解决方案设计提示词

```markdown
请设计针对以下问题的解决方案：

**问题**：{填写问题}

**设计约束**：
- 技术栈：{填写技术栈要求}
- 成本限制：{填写成本限制}
- 时间要求：{填写时间要求}
- 质量要求：{填写质量要求}

**设计维度**：
1. **架构设计**：整体架构和组件设计
2. **技术选型**：具体的技术和工具选择
3. **实施步骤**：详细的实施步骤和时间线
4. **测试方案**：质量保证和测试方案
5. **部署方案**：部署和运维方案
6. **监控方案**：监控和告警方案

**输出要求**：
- 提供架构图描述（文字或Mermaid）
- 详细的技术选型理由
- 分阶段的实施计划
- 具体的代码示例或配置示例
- 风险评估和应对策略
```

### 四、对比分析提示词

```markdown
请对比分析以下方案：

**对比对象**：{填写对比对象，如：方案A vs 方案B}

**对比维度**：
1. **功能对比**：核心功能的完整性和易用性
2. **性能对比**：响应时间、吞吐量、资源消耗
3. **成本对比**：开发成本、运维成本、使用成本
4. **扩展性对比**：未来扩展的难易程度
5. **安全性对比**：数据安全和系统安全
6. **生态对比**：社区支持和工具生态
7. **学习曲线**：上手难度和学习成本

**输出要求**：
- 使用对比表格清晰展示差异
- 每个维度给出具体的评分（1-5分）
- 提供选择建议和适用场景
- 给出具体的迁移方案（如适用）
```

### 五、实施计划提示词

```markdown
请制定以下方案的实施计划：

**方案名称**：{填写方案名称}

**计划要求**：
- 总时间：{填写总时间要求}
- 里程碑：{填写关键里程碑}
- 资源：{填写可用资源}
- 风险：{填写已知风险}

**计划维度**：
1. **阶段划分**：将实施过程划分为几个阶段
2. **阶段目标**：每个阶段的具体目标和产出
3. **时间安排**：每个阶段的时间安排和依赖关系
4. **资源分配**：人力、物力、财力的分配
5. **风险管理**：风险识别、评估和应对措施
6. **质量保证**：质量标准和验收标准
7. **沟通计划**：沟通频率、方式和内容

**输出要求**：
- 使用甘特图描述时间安排（文字描述）
- 详细的资源分配表
- 具体的风险管理计划
- 明确的验收标准
- 可执行的行动项清单
```

### 六、案例总结提示词

```markdown
请总结以下案例：

**案例名称**：{填写案例名称}

**总结维度**：
1. **背景介绍**：案例的背景和起因
2. **问题分析**：遇到的问题和挑战
3. **解决方案**：采用的解决方案
4. **实施过程**：具体的实施步骤和关键决策
5. **成果评估**：实施后的成果和效果
6. **经验教训**：成功的经验和失败的教训
7. **改进建议**：未来的改进方向和建议
8. **可复用组件**：可以复用的组件或模式

**输出要求**：
- 结构化的案例描述
- 量化的成果评估
- 具体的经验教训
- 可操作的建议
- 可复用的模板或代码
```

## 🔄 使用流程

### 一、标准调研流程

1. **准备阶段**
   - 确定调研目标
   - 选择合适模型
   - 准备背景信息
   - 选择提示词模板

2. **执行阶段**
   - 使用准备提示词初始化
   - 进行问题分析
   - 设计解决方案
   - 制定实施计划

3. **输出阶段**
   - 整理调研结果
   - 生成标准报告
   - 提供行动建议
   - 归档调研文档

4. **优化阶段**
   - 收集使用反馈
   - 优化提示词
   - 更新模板库
   - 分享最佳实践

### 二、快速调研流程

1. **明确问题**（5分钟）
   - 一句话描述问题
   - 确定核心需求
   - 设定时间限制

2. **快速分析**（10分钟）
   - 使用快速调研模板
   - 聚焦核心问题
   - 提供可行方案

3. **输出建议**（5分钟）
   - 简洁的结论
   - 具体的建议
   - 明确的行动项

### 三、对比分析流程

1. **确定对比项**（5分钟）
   - 明确对比对象
   - 设定对比维度
   - 确定评价标准

2. **收集信息**（15分钟）
   - 收集各方案信息
   - 整理关键数据
   - 识别核心差异

3. **分析对比**（10分钟）
   - 使用对比模板
   - 量化分析差异
   - 给出选择建议

4. **输出报告**（5分钟）
   - 生成对比报告
   - 提供决策支持
   - 归档分析结果

## 📈 质量评估标准

### 一、内容质量标准

| 维度 | 优秀标准 | 合格标准 | 不合格表现 |
|------|----------|----------|------------|
| **完整性** | 覆盖所有关键维度，无遗漏 | 覆盖主要维度，少量遗漏 | 关键信息缺失 |
| **准确性** | 信息准确无误，有数据支持 | 信息基本准确，少量错误 | 信息错误较多 |
| **深度** | 深入分析根本原因和影响 | 分析有一定深度 | 分析停留在表面 |
| **实用性** | 提供可落地的具体方案 | 方案基本可行 | 方案不可行 |
| **创新性** | 提供创新思路和方案 | 方案合理但常规 | 方案陈旧无效 |

### 二、格式质量标准

| 维度 | 优秀标准 | 合格标准 | 不合格表现 |
|------|----------|----------|------------|
| **结构** | 结构清晰，逻辑严谨 | 结构基本清晰 | 结构混乱 |
| **可读性** | 易于理解，表达清晰 | 基本可读 | 难以理解 |
| **规范性** | 符合模板要求，格式规范 | 基本符合要求 | 格式混乱 |
| **可视化** | 合理使用表格、图表 | 少量可视化元素 | 无可视化 |
| **完整性** | 包含所有必要部分 | 主要部分完整 | 部分缺失 |

### 三、时效性标准

| 类型 | 优秀 | 合格 | 不合格 |
|------|------|------|--------|
| **快速调研** | 20分钟内完成 | 30分钟内完成 | 超过30分钟 |
| **标准调研** | 2小时内完成 | 4小时内完成 | 超过4小时 |
| **深度调研** | 1天内完成 | 2天内完成 | 超过2天 |

## 🚀 最佳实践

### 一、提示词优化技巧

1. **明确具体**：避免模糊描述，提供具体信息
2. **分步引导**：将复杂问题分解为多个步骤
3. **提供示例**：给出期望输出的示例
4. **设定约束**：明确时间、成本、技术等约束
5. **要求结构化**：要求使用特定格式输出

### 二、模型选择策略

1. **推理分析**：选择Gemini、Claude等强推理模型
2. **代码生成**：选择DeepSeek、GPT等编程强模型
3. **中文场景**：选择GLM、DeepSeek等中文优模型
4. **成本敏感**：选择DeepSeek、硅基流动等低成本模型
5. **创新需求**：选择Claude、GPT等创意强模型

### 三、效率提升技巧

1. **模板化**：建立常用模板库
2. **批处理**：批量处理类似问题
3. **自动化**：自动化文档生成和归档
4. **知识复用**：建立案例库和最佳实践库
5. **协作优化**：多人协作，分工明确

### 四、质量控制技巧

1. **交叉验证**：用不同模型验证同一问题
2. **人工审核**：关键文档人工审核
3. **持续优化**：根据反馈持续优化提示词
4. **标准化**：建立统一的质量标准
5. **工具辅助**：使用工具检查格式和内容
