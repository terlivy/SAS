## 一、迭代背景

### 1.1 触发原因

v1.4 存在以下核心问题，驱动升级到 v1.5：

| 问题 | 现状（v1.4） | 目标（v1.5） |
|------|------------|------------|
| 用户参与度过高 | 决策点分散，用户频繁被打断 | 用户只参与计划+方案两个决策点 |
| Agent 能力无差异 | 所有 Agent 相同职责 | 按能力档案分配任务 |
| 任务复杂度无分级 | 统一流程，复杂任务效率低 | L1/L2/L3 分级，差异化流程 |
| 权限边界模糊 | "不擅自做主"无法量化 | L0-L3 精确权限矩阵 |

### 1.2 升级目标

建立 **CEO 模式协作架构**：
- 用户 = CEO，只审批计划和方案
- SC = PM，统筹协调
- Leader = 项目经理，自治决策
- Worker = 执行者，专注执行

## 四、L1/L2/L3 复杂度分级定义

### L1 — 简单任务（快速通道）

**判断标准**（满足任一）：
- 单 Agent 可完成
- 执行步骤 ≤ 3
- 预估工时 ≤ 30 分钟
- 无跨系统依赖

**流程**：阶段1 → 阶段3（简化审批）→ 阶段5 → 阶段6

**示例**：查天气、格式转换、简单问答

### L2 — 中等任务（标准流程）

**判断标准**（满足任一）：
- 需 2 个 Agent 协作
- 执行步骤 4-6
- 预估工时 30 分钟 - 2 小时
- 有 1-2 个跨系统依赖

**流程**：六阶段完整流程，方案需用户确认

**示例**：文档编写、简单前后端开发、数据分析报告

### L3 — 复杂任务（完整流程）

**判断标准**（满足任一）：
- 需 3+ Agent 协作
- 执行步骤 > 6
- 预估工时 > 2 小时
- 有 3+ 跨系统依赖
- 涉及外部 API / 数据库 / 部署

**流程**：六阶段完整 + Leader 自治协调 + 详细任务成绩单

**示例**：监控系统搭建、多系统集成项目
